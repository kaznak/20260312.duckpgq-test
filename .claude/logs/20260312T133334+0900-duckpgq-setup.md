# DuckPGQ テスト環境セットアップ

作業日時: 20260312T133334+0900
担当: Claude Opus 4.6

## 作業内容

DuckPGQ（DuckDB の SQL/PGQ 拡張）を試すための環境を構築した。

## 判断・決定

- Nix flake + direnv による環境構築を採用（ユーザー指示）。
- nixpkgs-unstable の DuckDB v1.4.4 を使用。DuckPGQ は community 拡張として `INSTALL duckpgq FROM community` でインストール可能。

## 発見・知見

- DuckPGQ は DuckDB v1.0.0 以降、community 拡張リポジトリから直接インストールできる。署名なしモードは不要。
- SQL/PGQ の構文は既存テーブルの上にプロパティグラフを定義し、`GRAPH_TABLE()` でグラフパターンマッチや最短経路クエリを実行する形式。
- プロパティグラフはビューに類似しており、元テーブルの変更が自動的に反映される。

## 成果物

- `flake.nix`: DuckDB を含む Nix 開発環境定義
- `.envrc`: direnv 設定
- `.gitignore`: `.direnv/` を除外
- `examples/getting-started.sql`: DuckPGQ の基本操作を確認する SQL スクリプト

## 動作確認結果

`examples/getting-started.sql` の各クエリが正常に動作することを確認済み。

- パターンマッチ: Alice の友達の友達 → (Bob→Charlie), (Bob→Diana), (Charlie→Diana)
- 最短経路: Alice→Diana = 2ホップ（Alice→Bob→Diana）

## 振り返り

- DuckPGQ のセットアップは簡単で、既存の DuckDB 環境に `INSTALL`/`LOAD` するだけで利用開始できる。
- SQL/PGQ 構文は直感的で、Cypher に似たパターンマッチ記法を SQL 内で使える点が特徴的。
- 今後、より複雑なグラフ（多種類の頂点・辺、重み付き経路探索、PageRank 等のアルゴリズム）を試す余地がある。

## 要約

Nix flake で DuckDB v1.4.4 環境を構築し、DuckPGQ 拡張のインストール・基本的なグラフクエリ（パターンマッチ、最短経路）の動作を確認した。動作確認手順は `examples/getting-started.sql` にまとめた。
